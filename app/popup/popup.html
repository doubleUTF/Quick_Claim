CPT List

 

  • Why is only Office Ally supported?
  • This is the beta and I happen to work on their site most often. Support for sites coming soon: Optum/United Health Care, Ability Network
  • How come only 12 rows can be filled on OA?
  • Currently when a row is added, the element id naming scheme doesn't properly increment. In other words, Office Ally dropped the ball.
  • Why do I get error messages when I input data into Claim Form section?
  • The jQuery Validator plugin validates data as you type. Finish the entire input and you should be fine.
  • Is this app safe to use?
  • Yes. Only CPT profile data is stored in localStorage, so only the local computer has access to it. No sensitive or identifying data is stored.
  • I found a bug/ want a new feature/ have questions.
  • Go to the Github page and submit an issue.

Directions

  1. Ensure you are on a supported site(Office Ally only in beta) and at the HCFA-1500 form section. The status bar on the Claim Form tab will indicate this.
  2. Fill out sections 1-21 either by loading saved patient data or entering new data.
  3. Go to the Profiles tab to setup and save profile data for commonly used CPT codes. These codes are not specific to any patient or site and can thus be used for any patient. You only need to do this once per CPT.
  4. Go to Claim Form tab, enter CPT codes and dates. Click Fill Form button and verify data is correct. Quick Claim will automatically provide each date of service for all CPT codes and all diagnosis for each CPT. If this is undesired, click Undo Fill and enter data manually.
